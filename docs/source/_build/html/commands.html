Optim3D

CLI tool to manage full optimized reconstruction of large-scale 3D building models.

Optim3D [OPTIONS] COMMAND [ARGS]...

index2d

QuadTree indexing and tiling of 2D building footprints.

Optim3D index2d [OPTIONS] [FOOTPRINTS]

Options

--output <output>

Output directory.

Default:

./output

--osm <osm>

Download and work with building footprints from OpenStreetMap [west, north, est, south].

--crs <crs>

Specify the Coordinate Reference System (EPSG).

--max <max>

Maximum number of buildings per tile.

Default:

3500

Arguments

FOOTPRINTS

Optional argument

index3d

OcTree indexing of 3D point cloud using Entwine.

Optim3D index3d [OPTIONS] POINTCLOUD

Options

--output <output>

Output directory.

Default:

./output

Arguments

POINTCLOUD

Required argument

post

Postprocess the generated CityJSON files.

Optim3D post [OPTIONS]

Options

--cityjson <cityjson>

CityJSON files directory.

Default:

./output/model/cityjson

reconstruct

Optimized 3D reconstruction of buildings using GeoFlow.

Optim3D reconstruct [OPTIONS]

Options

--pointcloud <pointcloud>

3D point cloud tiles directory.

Default:

./output/pointcloud_tiles

--footprints <footprints>

2D building footprints tiles directory.

Default:

./output/footprint_tiles

--output <output>

Output directory.

Default:

./output

tiler3d

Tiling of point cloud using the calculated processing areas.

Optim3D tiler3d [OPTIONS]

Options

--areas <areas>

The calculated processing areas.

Default:

./output/processing_areas.gpkg

--indexed <indexed>

Indexed 3D point cloud directory.

Default:

./output/indexed_pointcloud

--output <output>

Output directory.

Default:

./output
